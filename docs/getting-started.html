Get Started

Install SWE_BUILDER, authenticate to your Copilot Studio environment, and ship your first agent. Two install paths cover Windows-only and cross-platform setups.

Prerequisites

The start.cmd path needs nothing beyond a Windows 11 machine with internet access. Everything else is auto-installed via winget. The npm path requires:

  • Node.js 18.x or newer
  • Python 3.10 or newer
  • Git 2.30 or newer
  • A Microsoft Entra account with access to a Copilot Studio environment
  • Power Platform CLI (auto-installed during first run if missing)
Recommended: Claude Code or GitHub Copilot CLI
SWE_BUILDER orchestrates Claude Code or GitHub Copilot CLI as the engineering brain. The embedded terminal will prompt you to install one if neither is present.

Option A: start.cmd (Windows, recommended)

Zero-prerequisite install for Windows 11. Uses winget. No admin rights required.

  1. Clone the repository

    git clone https://github.com/dayour/swe_builder.git
cd swe_builder

    If git is not installed, download a zip from GitHub and extract it.

  2. Run the launcher

    .\start.cmd

    On first run, the launcher detects missing dependencies and installs them via winget. Expect a few minutes for Node, Python, Git, and Claude Code. Subsequent runs skip install and start in roughly one second.

  3. Wait for the dashboard

    The launcher starts the FastAPI backend, the React dashboard, and the embedded terminal server. A browser window opens automatically at http://localhost:8000. If port 8000 is in use, the launcher falls back to the next available port and updates CORS automatically.

Option B: npm install (cross-platform)

Works on macOS, Linux, and Windows. Requires Node and Python pre-installed.

  1. Install globally

    npm install -g swe_builder

    The post-install hook builds the frontend with Vite, runs pip install for backend dependencies, and seeds environment variables.

  2. Start the app

    swe_builder start

  3. Open the dashboard

    Navigate to http://localhost:8000 if the browser does not open automatically.

First-run flow

The dashboard launches the embedded Claude terminal automatically. Claude greets you and asks for context.

  1. Pick an account

    If you have multiple GitHub or Microsoft accounts, the terminal asks which one to use. Picks are persisted to tools/session-config.json.

  2. Pick a Copilot Studio environment

    Claude calls pac auth list and presents your environments. The default environment is persisted at the session level so you only answer this once.

  3. Confirm and proceed

    You are ready to invoke any of the 13 workflow skills. Use the dashboard buttons or type slash commands directly.

Your first project

Run these in the embedded terminal, in order. Each one moves the project forward through one phase of the workflow.

  1. Create the project

    /mcs-init MyFirstAgent

    Creates Build-Guides/MyFirstAgent/ with a docs/ folder for SDR documents and a starter brief.json.

  2. Drop in your requirements

    Place SDR documents (Word, Markdown, or PDF) into Build-Guides/MyFirstAgent/docs/, or paste requirements directly into chat.

  3. Research

    /mcs-research MyFirstAgent

    Reads documents, identifies agents, designs architecture (single vs. multi), enriches brief.json, and generates eval sets.

  4. Review the brief in the dashboard

    The brief editor shows every section. Answer open questions, edit instructions, add or remove tools, then save.

  5. Build

    /mcs-build MyFirstAgent agent-name

    Constructs the agent in MCS via the hybrid stack. Eval-driven loop validates as it goes. Publishes when all gates pass.

  6. Generate the report

    /mcs-report MyFirstAgent agent-name --type customer

    Produces a customer-shareable Markdown report.

Troubleshooting

Port 8000 in use

The launcher auto-detects the conflict and falls back to the next free port. CORS origins update dynamically. Look for the actual port in the terminal output.

winget command not found

Install App Installer from the Microsoft Store, then re-run start.cmd. winget ships with Windows 11 by default; older Windows 10 builds need the manual install.

PAC CLI auth fails

Run pac auth clear then pac auth create. If MFA blocks the device flow, use --deviceCode.

Build hangs at browser step

Ensure no other Edge instance is bound to remote debugging port 9222. Kill stale instances or run tools\start-edge-debug.cmd manually.

YAML validation errors

Run tools\om-cli\om-cli.exe validate Build-Guides\Project\agents\name\topics\topic.yaml to see field-level errors with line numbers.

Eval scores below threshold

Use /mcs-fix Project agent-name. QA Challenger classifies failures and Prompt or Topic Engineer applies fixes automatically.
