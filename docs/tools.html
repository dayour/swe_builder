Tools

Standalone CLI tools that power SWE_BUILDER. They live in tools/ and are invoked by skills as needed. Each one does one thing well and writes structured output that the next layer consumes.

Validation pipeline

Four tools, four layers. Run them in order before pasting topic YAML into Copilot Studio.

om-cli (ObjectModel CLI)

What it does: Full YAML validation against the official MCS authoring schema. Knows 357 concrete types and 433 kind values. Catches unknown nodes and missing required fields with line-level error reporting.

Built on: .NET 10 + Microsoft Bot Framework Composer ObjectModel. Auto-updates from upstream via the pre-push git hook.

# Validate a topic
.\tools\om-cli\om-cli.exe validate Build-Guides\MyAgent\agents\name\topics\OrderStatus.yaml

# Search the schema
.\tools\om-cli\om-cli.exe search "AdaptiveCard"

# Show field hierarchy for a type
.\tools\om-cli\om-cli.exe hierarchy Microsoft.AdaptiveDialog

gen-constraints.py

What it does: Pre-generation constraint extraction. Queries om-cli for required fields per node type and emits structured JSON the Topic Engineer uses to generate correct YAML on the first try.

python tools/gen-constraints.py Microsoft.AdaptiveDialog Microsoft.OnIntent

semantic-gates.py

What it does: Five semantic validation gates beyond structural checks.

  1. PowerFx gate: Validates expressions against the official catalog (tools/powerfx-catalog.json).
  2. Cross-reference gate: Confirms every topic:Name, variable:X, and tool:Y reference resolves.
  3. Variable flow gate: Catches reads-before-writes and orphaned writes.
  4. Channel compatibility gate: Flags unsupported card features per channel.
  5. Connector schema gate: Verifies action inputs match the connector's declared schema.
python tools/semantic-gates.py topics/OrderStatus.yaml --brief Build-Guides/MyAgent/agents/name/brief.json

drift-detect.py

What it does: Compares brief specs vs. built YAML. Flags missing topics, extra topics, trigger phrase mismatches, and variable drift between the source of truth and the actual artifact.

python tools/drift-detect.py Build-Guides/MyAgent/agents/name/brief.json

schema-lookup.py (legacy fallback)

What it does: Kind-value validation. Lighter-weight than om-cli, used when .NET 10 is not available. Confirms a kind value exists in the schema (433 values).

python tools/schema-lookup.py Microsoft.AdaptiveDialog

Evaluation tools

direct-line-test.js

What it does: Direct Line API test runner. Auto-fetches tokens via the Token Endpoint, retries with backoff, 60s timeout per turn, structured partial results on failure.

node tools/direct-line-test.js --brief Build-Guides/MyAgent/agents/name/brief.json --set critical

eval-scoring.js

What it does: Standalone eval scoring engine. Decoupled from the test runner so any transcript can be re-scored against any rubric. Supports semantic similarity, exact match, regex, contains, must-not-contain, latency threshold, and intent classification.

node tools/eval-scoring.js --transcript run.json --rubric brief.json

multi-model-review.js

What it does: Optional second-opinion GPT review via the GitHub Copilot API. Used for instructions, topics, briefs, and flows. Automatically invoked in /mcs-build (review-instructions, review-topics) and /mcs-report. Exits silently if not configured. Never blocks the build.

gh auth login
node tools/multi-model-review.js --target instructions --file brief.json

Lifecycle tools

replicate-agent.js

What it does: Clone an agent's full configuration from one environment to another. Used by /mcs-deploy in agent mode for fast dev to test promotion.

solution-library.js

What it does: Browse, download, analyze, and upload to the team SharePoint solution library. 30 solutions are pre-cached locally with an index.

upstream-check.js

What it does: Detect drift against tracked upstream repos (configured in knowledge/upstream-repos.json). Surfaces breaking changes in MS Learn samples and the official ObjectModel.

flow-composer.js (lib)

What it does: Assembles Power Automate flow JSON from templates. Used by flow-manager.js as the composition step in the spec-to-JSON-to-create pipeline.

mcs-lsp.js

What it does: Language Server Protocol client for workspace pull and push. Lets Copilot Studio's authoring service validate YAML in-line during editing.

dataverse-helper.ps1

What it does: PowerShell Dataverse Web API helper. Wraps token acquisition and HTTP calls for instructions update, knowledge upload, and security settings.

Git hooks

Two hooks live in tools/git-hooks/ and are installed by start.js:

  • pre-commit: Core file protection. Blocks commits that touch protected paths without explicit override.
  • pre-push: Auto-updates om-cli from the upstream ObjectModel source.
