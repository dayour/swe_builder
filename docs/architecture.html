Architecture

SWE_BUILDER is a multi-process workstation app: a FastAPI backend, a React + Vite dashboard, and a Node-pty WebSocket terminal that hosts Claude Code as the operator. Underneath, a hybrid stack chooses the right tool for each step against Microsoft Copilot Studio.

Agent Teams

You speak to the lead orchestrator. The lead delegates to seven specialists, each with its own context window. Specialists challenge each other. The lead executes only validated artifacts against the hybrid stack.

Multi-agent topology

When research scores three or more multi-agent factors, the orchestrator delegates to specialist children. Children are built and evaluated first, then the orchestrator wires them via child-agent connections.

flowchart TB User([User]) --> Orch[Orchestrator Agent] Orch --> A[Sales Specialist] Orch --> B[Support Specialist] Orch --> C[Knowledge Specialist] A --> ToolsA[Tools A] B --> ToolsB[Tools B] C --> KB[(Knowledge sources)]

Six factors decide single vs. multi-agent (see knowledge/frameworks/architecture-scoring.md):

  1. Distinct user personas with different needs
  2. More than five tools or connectors
  3. More than two knowledge domains
  4. Conflicting tone or response style requirements
  5. Independent workflows that compose later
  6. Different security or data boundaries
