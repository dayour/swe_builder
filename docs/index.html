Microsoft Copilot Studio Automation

Build production agents end-to-end with one CLI.

SWE_BUILDER automates the full Copilot Studio agent lifecycle: data intake, research, architecture, build, evaluation, and an automated fix loop. A hybrid stack combines PAC CLI, Dataverse API, Code Editor YAML, Direct Line, and Playwright, coordinated by Claude Code with seven AI teammates that peer-review every step.

What it is

SWE_BUILDER is a multi-process workstation app for building Microsoft Copilot Studio agents at production quality. Customer documents go in. A published agent backed by an eval suite and a customer-shareable build report come out. Claude Code is the operator. You watch the work happen in an embedded terminal.

Brief-Driven

One source of truth

Every project has a brief.json that captures architecture, instructions, tools, model, knowledge, topics, and eval sets. Build, eval, fix, and reports all flow from it.

Eval-Verified

Quality bar enforced

Eval sets are generated from the spec and run through Direct Line API. Critical gate, per-capability iteration, and regression suite must pass before publish.

Hybrid Stack

Right tool, right job

PAC CLI for lifecycle, Dataverse API for config, Code Editor YAML for topics, Direct Line for tests, Playwright only for UI-only operations.

AI Teammates

Built-in peer review

Seven specialist sub-agents (research, prompt, topic, QA, repo-checker, flow-designer, repo-optimizer) challenge each other before any change reaches Copilot Studio.

Multi-Agent First

Decompose by default

Six-factor objective scoring of the use case decides single vs. multi-agent. Children are built first, then the orchestrator wires them.

Continuous Learning

Every build improves the next

A learnings index captures insights with confirmed counts and staleness flags. Future research consults them without overriding fresh decisions.

Quickstart

Two paths. Pick whichever matches your environment.

Option A: start.cmd (Windows, zero prerequisites)

git clone https://github.com/dayour/swe_builder
cd swe_builder
.\start.cmd

On first run, start.cmd uses winget (built into Windows 11) to install Node.js, Python, Git, Claude Code, and dependencies. No admin rights required. Subsequent runs detect everything is present and launch in roughly a second.

Option B: npm install (cross-platform)

npm install -g swe_builder
swe_builder start

Requires Node.js 18+ and Python 3.10+ already installed. The post-install hook builds the frontend, installs Python deps, and configures environment variables.

First-run setup
When the embedded Claude terminal opens, Claude asks for your tenant account, Copilot Studio environment, and runs pac auth create. Your selection persists across the session in tools/session-config.json.

Workflow overview

Six phases drive every project. Skills are invoked by name from the dashboard buttons or directly from Claude Code with slash commands.

  1. Init & Context

    /mcs-init creates a project folder. Optional /mcs-context CustomerName uses WorkIQ MCP to pull every email, meeting, document, and Teams chat related to the customer and pre-fills 60 to 80 percent of research.

  2. Research

    /mcs-research reads documents, identifies agents, researches MCS components across MS Learn, web, and the live UI, then produces a fully enriched brief.json with five eval sets (critical, functional, integration, conversational, regression).

  3. Build

    /mcs-build uses the hybrid stack to construct each agent. An eval-driven loop inside the build runs the critical gate, per-capability iteration, and regression suite. Failures are auto-fixed by the prompt and topic engineers.

  4. Evaluate

    /mcs-eval runs eval sets via Direct Line API (with a Playwright Test Chat fallback and an optional native MCS eval mode). Per-set pass rates are written back to brief.json.

  5. Fix

    /mcs-fix handles post-deployment issues. QA Challenger classifies failures into root causes (instruction gap, boundary violation, routing failure, knowledge gap, scoring issue), then PE and TE patch in parallel.

  6. Deploy & Report

    /mcs-deploy replicates dev to prod (agent or solution mode). /mcs-report renders customer-shareable summaries (brief, build, customer, deployment).

AI teammates

You speak to the lead orchestrator. Behind it, seven specialists collaborate, debate, and execute. The lead delegates and coordinates; teammates challenge each other and produce validated artifacts; the lead executes them via the hybrid stack.
