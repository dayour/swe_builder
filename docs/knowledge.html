Knowledge System

Four layers feed every architectural decision. Each layer answers a different question. Together they make every build smarter than the last.

Cache: 18 quick-reference cheat sheets

Cached inventories of MCS capabilities. Each file has freshness metadata. Tiered by criticality.

Tier 1 (build-critical)

Auto-refreshed at session start if older than 7 days. Refresh failures block the build.

  • triggers.md — every trigger type with config and gotchas
  • models.md — every available model with default and cost
  • mcp-servers.md — every MCP server (preview, GA) with capabilities
  • connectors.md — connector catalog with auth requirements
  • knowledge-sources.md — supported knowledge types and their limits
  • channels.md — every channel with feature support matrix

Tier 2 (build-phase)

Refreshed before /mcs-build if stale. Warns but does not block.

  • api-capabilities.md — what each layer of the hybrid stack can do
  • instructions-authoring.md — instruction patterns and antipatterns
  • generative-orchestration.md — how the orchestrator routes
  • adaptive-cards.md — card features by channel
  • ai-tools-computer-use.md — preview AI tool capabilities
  • power-automate-integration.md — flow patterns and limits

Tier 3 (reference)

Refreshed on demand via /mcs-refresh. Stable enough that monthly is fine.

  • eval-methods.md — every test method with examples
  • security-auth.md — auth patterns and managed identity
  • agent-lifecycle.md — create, publish, deploy, retire
  • limits-licensing.md — quotas and license requirements
  • powerfx-variables.md — variable types and PowerFx functions
  • conversation-design.md — turn structure and disambiguation

Patterns: stable HOW-TO references
